# Community Code of Conduct

万象灵枢希望成为一个对人类开发者和 Agent 实验者都友好的山门。

我们默认每一个来到这里的人，都是为了把项目修得更稳、更强、更有意思。  
为了让协作可以长期持续，请共同遵守下面这些基本规则。

## 我们鼓励的行为

- 友善、尊重、具体地表达不同意见
- 把批评落在代码、流程、验证和事实，而不是人身上
- 给出上下文、复现路径、日志和验收方式，帮助别人理解问题
- 对新手保持耐心，对复杂问题保持诚实
- 尊重不同背景、经验、语言和认知方式
- 让协作更利于 OpenClaw 和整个系统闭环，而不是制造额外门槛

## 不被接受的行为

- 人身攻击、羞辱、嘲讽、侮辱性语言
- 骚扰、威胁、跟踪、恶意挑衅
- 歧视性言论，包括但不限于性别、性取向、种族、地区、宗教、残障、年龄等
- 故意公开他人的隐私信息、令牌、密钥或安全细节
- 明知会破坏仓库协作秩序，仍持续刷屏、引战或恶意消耗维护者精力
- 借“玩梗”“测试”“研究”为名发布可直接利用的攻击内容

## 项目语境下的额外要求

这是一个生产级 Agent 项目，不是纯 demo 仓库。

因此我们特别不欢迎：

- 把人类重新塞回本应由 OpenClaw 自主完成的主线
- 为了“好看”而引入不真实的演示闭环
- 在未说明的情况下提交生产凭据、真实用户数据或运维秘密
- 借 issue / PR 公开披露高危安全细节

## 执行责任

维护者有责任：

- 解释社区规则
- 对明显越界行为作出提醒、编辑、隐藏、锁定或移除
- 在必要时拒绝不适合继续参与的账号或内容

## 举报方式

如果你遇到不当行为，请不要在公开 issue 里扩大伤害。

推荐方式：

1. 优先通过仓库所有者 `@usernema` 的私下联系渠道反馈
2. 如果问题涉及安全、隐私或凭据泄露，请同时参考 `/Users/mac/A2Ahub/SECURITY.md`
3. 如果举报对象本身是仓库 owner，建议直接使用 GitHub 平台的举报 / 支持渠道

请在举报中尽量包含：

- 发生了什么
- 相关链接
- 截图或记录
- 你希望得到的处理方式

## 处理原则

我们会尽量：

- 认真对待每次举报
- 尽可能保护举报者隐私
- 根据影响程度采取比例合适的处理

可能的处理包括：

- 口头提醒
- 编辑 / 删除内容
- 锁定讨论
- 暂时或永久限制参与

## 适用范围

本准则适用于：

- 仓库 issue
- Pull Request
- code review
- 文档讨论
- 与本项目直接相关的公开或半公开协作空间

## 最后一句

我们希望这座山门既有锋芒，也有分寸。  
可以激烈讨论，但不要彼此伤害。
